Why Wearables Are Different

Unique Challenges of Wearable Design

Constraints

  • Tiny screen (38-45mm)
  • Limited battery life
  • Brief interaction time
  • No keyboard input
  • Worn in public spaces

Opportunities

  • Always with the user
  • Quick glances suffice
  • Haptic feedback available
  • Health sensors built-in
  • Personal & intimate device

Context of Use: When & How

  • Users glance at their watch for 2-5 seconds on average
  • Often used while doing something else (walking, exercising)
  • Information must be immediately comprehensible
  • Interactions must be completable in seconds, not minutes
  • Users expect instant feedback and response

7 Design Principles for Wearables

1

Glanceability

Information should be visible and understandable at a quick glance. Users shouldn't need to study the screen to get the information they need.

EXAMPLE

Progress rings that show completion percentage visually. A user can see they're at 75% of their water goal without reading any numbers.

2

Keep It Simple

Limit functionality to what's essential. Every feature must earn its place on the tiny screen. Focus on doing a few things exceptionally well.

EXAMPLE

Our app tracks only calories and water - not protein, carbs, fiber, vitamins, etc. Two focused features instead of ten mediocre ones.

‚ú®
3

Minimalistic Interface

Use sharp contrast, basic typography, and clean layouts. Remove every element that isn't absolutely necessary for the user's task.

EXAMPLE

Dark background with bright orange/cyan accents. Large, bold numbers. No decorative elements or complex graphics.

üéØ
4

Quick Interactions

No complex gestures or multi-step processes. Every action should be completable with one or two taps. Pre-populated options beat manual input.

EXAMPLE

Quick-add buttons with preset values (+100, +200, +500) instead of requiring users to type numbers.

‚ö°
5

Haptic Feedback

Physical vibration provides confirmation without requiring visual attention. Users know their action succeeded through touch.

EXAMPLE

A subtle 'tap' when adding an entry, and a celebratory vibration pattern when reaching a daily goal.

üì≥
6

Offline-First Design

Wearables often have limited or no connectivity. Core features must work without internet. Sync when possible, fail gracefully when not.

EXAMPLE

All entries stored locally on the watch. Motivational quotes have offline fallbacks. Data never lost due to connectivity issues.

üì¥
7

Personalization

Allow users to customize goals and preferences. A personalized experience provides more value and keeps users engaged.

EXAMPLE

Users can set their own daily calorie and water goals based on their personal health objectives.

App Architecture: MVVM Pattern

Structure

  • Models - Data structures
  • Views - SwiftUI interfaces
  • ViewModels - Business logic
  • Services - Storage, API, Haptics

Benefits

  • Separation of concerns
  • Testable components
  • Reusable services
  • Clean, maintainable code

Technologies We'll Use

  • SwiftUI - Modern declarative UI framework
  • WatchKit - WatchOS-specific APIs (haptics)
  • UserDefaults - Simple key-value storage
  • URLSession - Network requests (async/await)
  • ObservableObject - Reactive data flow
