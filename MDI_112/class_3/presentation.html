What is Core Motion?

Core Motion Framework

What It Does

  • Accelerometer data
  • Gyroscope data

  • Pedometer & activity data

Motion Sensors Explained

üìê Accelerometer

Measures acceleration forces (including gravity). Tells you device orientation and movement speed.

üîÑ Gyroscope

Measures rotation rate around axes. Detects how fast device is spinning.

üß≠ Magnetometer

Measures magnetic field strength. Used for compass heading.

üëü Pedometer

Counts steps, distance, floors climbed. Uses motion coprocessor.

Understanding Acceleration Values

Gravity = 1G = 9.8 m/s¬≤

When device is flat on table, Z axis reads approximately -1.0 (gravity pulling down)

Device Flat (face up)

X: 0.0, Y: 0.0, Z: -1.0

Device on Left Edge

X: -1.0, Y: 0.0, Z: 0.0

Device Standing Up

X: 0.0, Y: -1.0, Z: 0.0

Device Shaking

Values > 1.0 or < -1.0

CMMotionManager

CMMotionManager Class

  • Central class for all motion data access
  • Create one instance per app (singleton pattern recommended)
  • Check sensor availability before using
  • Configure update interval (how often to get data)
  • Start/stop updates as needed to save battery

üí° Core Motion doesn't require special permissions like HealthKit. Just import and use!

Basic Setup

import CoreMotion class MotionManager: ObservableObject { private let motionManager = CMMotionManager() init() { // Check if accelerometer is available if motionManager.isAccelerometerAvailable { print("Accelerometer is available!") } // Set how often we want updates (in seconds) motionManager.accelerometerUpdateInterval = 0.1 // 10 times per second } }

Two Ways to Get Data

Push Updates (Recommended)

Start updates with a handler closure. Data pushed to you automatically.

startAccelerometerUpdates(to:withHandler:)

Pull Updates

Start updates, then poll the accelerometerData property when needed.

startAccelerometerUpdates()
motionManager.accelerometerData

Starting Accelerometer Updates

func startUpdates() { guard motionManager.isAccelerometerAvailable else { return } motionManager.startAccelerometerUpdates( to: OperationQueue.main ) { [weak self] data, error in guard let data = data, error == nil else { return } // Access the acceleration values let x = data.acceleration.x let y = data.acceleration.y let z = data.acceleration.z // Update your UI! self?.xValue = x self?.yValue = y self?.zValue = z } }

Stopping Updates (Important!)

func stopUpdates() { motionManager.stopAccelerometerUpdates() }
‚ö†Ô∏è BATTERY WARNING

Always stop updates when not needed! Continuous sensor access drains battery quickly. Stop in onDisappear or when user isn't looking.

Choosing Update Interval

0.1s (10 Hz)

Good for UI updates, smooth animations

Recommended

0.05s (20 Hz)

Smoother but more battery drain

Games

0.5s (2 Hz)

Battery friendly, less smooth

Background

0.01s (100 Hz)

Very precise, high battery use

SwiftUI Integration

Motion Manager with SwiftUI

import CoreMotion import SwiftUI class MotionManager: ObservableObject { @Published var x: Double = 0 @Published var y: Double = 0 @Published var z: Double = 0 private let motion = CMMotionManager() func start() { motion.accelerometerUpdateInterval = 0.1 motion.startAccelerometerUpdates(to: .main) { data, _ in guard let data = data else { return } self.x = data.acceleration.x self.y = data.acceleration.y self.z = data.acceleration.z } } }

Using Motion Data in Views

struct MotionView: View { @StateObject private var motion = MotionManager() var body: some View { VStack { Text("X: \(motion.x, specifier: "%.2f")") Text("Y: \(motion.y, specifier: "%.2f")") Text("Z: \(motion.z, specifier: "%.2f")") } .onAppear { motion.start() } .onDisappear { motion.stop() } } }
